DB2 Monitor, monitor statements

By Sword Sun

發表於 2016-01-25

DB2 Monitor Statements

1)查看监视器选项
db2 get monitor switches

2)开启语句跟踪选项
db2 update monitor switches using statement on

3)创建语句监视器，结果输出到文件夹
mkdir /Users/swordsun/www/state_event
db2 “create event monitor state_event for statements write to file ‘/Users/swordsun/www/state_event’”

4)激活监视器
db2 set event monitor state_event state=1

5)运行应用程序 例如查询: select * from table_name;

6)完成输出
db2 flush event monitor state_event buffer

7)关闭、删除监视器
db2 set event monitor state_event state=0
db2 update monitor switches using statement off
db2 drop event monitor state_event

8)格式化结果并输出到一个文件
db2evmon -path /Users/swordsun/www/state_event > /Users/swordsun/www/query.sql

Other commands

  • db2 reset monitor all # reset monitor for all or for one db (name)
  • SELECT * FROM TABLE( SNAPSHOT_DYN_SQL( ‘db_sc’, -1 )) as SNAPSHOT_DYN_SQL # check snapshot in db

Bluemix Quick Start

Create an IBM id

https://www.ibm.com/account/us-en/signup/register.html
Ignore if you already have IBM id.

Sign Up or Log in Bluemix

  1. Bluemix public https://console.ng.bluemix.net/
  2. IBM Bluemix DevOps Services https://hub.jazz.net/

Create application

  1. Click CREATE APP -> WEB -> PHP and fill one app name (need to be unique)
  2. Read start coding docuemnt (CF and GIT)
  3. Click ‘Overview’ in left navigation.
  4. Click ‘ADD GIT’ then ‘CONTINUE’ to create git repo. ‘ADD GIT’ is now replaced with GIT URL and EDIT CODE button.
  5. Copy ‘Routes’ url appname.mybluemix.net and open with browser

Use Jazz to manange (Bluemix DevOps Services)

  1. Right click in left navigation and create a new file: info.php
  2. Put <?php phpinfo(); ?> in info.php
  3. Click ‘Git Repository’, select info.php and commit, then push the commit
  4. Go back to Edit page and click ‘Deploy’ icon (next to app status bar)
  5. Open http://appname.mybluemix.net/info.php in browser

Use local Git to manage

  1. Check email from sent from IBM DevOps Services (hub@jazz.net) and get Git Repository URL: https://hub.jazz.net/git/username/appname
  2. git clone https://hub.jazz.net/git/username/appname
  3. Create a new file: test.php with <?php echo ‘test’; ?>
  4. git add .
  5. git commit -m 'add test.php'
  6. git push origin master
  7. Open http://appname.mybluemix.net/test.php in browser

Test Stage

  1. Go to ‘BUILD & DEPLOY’ page and click ‘ADD STAGE’
  2. Change name to Test Stage. INPUT tab -> Input type NONE
  3. JOBS tab -> ADD JOB Test

  4. Put following code into Test Command

  5. SAVE

  6. Drag Test Stage and put before Deploy Stage.
  7. Click ‘Run Stage’ icon to test.

  1. Sign in your github, click ‘+’ in top right corner and select ‘New repository’.
